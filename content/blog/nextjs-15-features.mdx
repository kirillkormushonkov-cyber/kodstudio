# Что нового в Next.js 15

App Router окончательно стал основным способом строить приложения на Next.js.
Серверные компоненты, потоковый рендеринг и Server Actions работают предсказуемо
даже на больших проектах. Turbopack теперь включён по умолчанию для `next dev`
и стабильно собирает пакеты в продакшен.

## Server Actions

Серверные действия позволяют объявлять обработчики форм прямо рядом с компонентом
без отдельного API-маршрута. Это сокращает количество кода и упрощает
типизацию между клиентом и сервером — оба слоя видят одну и ту же сигнатуру.

## Кэш и ревалидация

Унифицированный кэш с гранулярной инвалидацией через теги стал предсказуемее.
Можно явно отметить, какие данные стоит держать, а какие — пересобирать после
каждого Server Action.

## Что это значит для проектов

Меньше кастомных слоёв между React и инфраструктурой, понятная модель кэширования
и ускоренный dev-цикл. Если вы давно откладывали миграцию на App Router — сейчас
самое время.
